## 1. Что такое Машинное Обучение?

**Машинное обучение (МО)** — это подраздел искусственного интеллекта, который изучает методы построения алгоритмов, способных обучаться на данных и делать прогнозы или принимать решения без явного программирования.

### Ключевая идея:
Вместо того чтобы писать правила для решения задачи, мы предоставляем алгоритму данные и позволяем ему самому найти закономерности.

### Традиционное программирование vs Машинное обучение:

| Традиционное программирование | Машинное обучение |
|-------------------------------|-------------------|
| Входные данные + Правила = Результат | Входные данные + Результат = Правила |
| Программист создаёт логику | Алгоритм находит закономерности |
| Жёсткие правила | Гибкая адаптация |

### Пример:
**Задача**: Определить, является ли письмо спамом.

- **Традиционный подход**: Программист пишет правила (если есть слово "выигрыш" → спам)
- **МО подход**: Алгоритм анализирует тысячи помеченных писем и сам находит признаки спама

## 2. Типы Машинного Обучения

### 1. Обучение с учителем (Supervised Learning)

Алгоритм обучается на размеченных данных (есть правильные ответы).

**Как работает:**
1. Предоставляем данные с известными ответами
2. Алгоритм находит закономерности
3. Используем модель для предсказания на новых данных

**Задачи:**
- **Классификация**: отнесение к категориям
  - Спам/не спам
  - Болен/здоров
  - Кошка/собака/птица
  
- **Регрессия**: предсказание числового значения
  - Цена дома
  - Температура завтра
  - Курс валют

**Алгоритмы:**
- Линейная регрессия
- Логистическая регрессия
- Деревья решений
- Метод опорных векторов (SVM)
- K-ближайших соседей (KNN)
- Нейронные сети

### 2. Обучение без учителя (Unsupervised Learning)

Алгоритм работает с неразмеченными данными (нет правильных ответов).

**Как работает:**
1. Предоставляем данные без меток
2. Алгоритм ищет скрытые структуры и закономерности

**Задачи:**
- **Кластеризация**: группировка похожих объектов
  - Сегментация клиентов
  - Группировка документов
  - Анализ социальных сетей
  
- **Понижение размерности**: упрощение данных
  - Визуализация многомерных данных
  - Сжатие данных
  
- **Поиск аномалий**: обнаружение необычных паттернов
  - Обнаружение мошенничества
  - Диагностика неисправностей

**Алгоритмы:**
- K-средних (K-means)
- Иерархическая кластеризация
- DBSCAN
- PCA (Principal Component Analysis)
- Автоэнкодеры

### 3. Обучение с подкреплением (Reinforcement Learning)

Алгоритм обучается через взаимодействие со средой, получая награды или штрафы.

**Как работает:**
1. Агент совершает действия в среде
2. Получает обратную связь (награда/штраф)
3. Корректирует стратегию для максимизации награды

**Компоненты:**
- **Агент**: обучающаяся система
- **Среда**: мир, в котором действует агент
- **Действия**: что может делать агент
- **Состояние**: текущая ситуация
- **Награда**: обратная связь от среды

**Применение:**
- Игры (AlphaGo, Dota 2)
- Робототехника
- Автономное вождение
- Управление ресурсами
- Торговые стратегии

## 3. Основные Алгоритмы Машинного Обучения

### 1. Линейная Регрессия

Предсказывает непрерывное значение на основе линейной зависимости.

**Формула:** `y = wx + b`

Где:
- `y` — предсказанное значение
- `x` — входное значение
- `w` — вес (коэффициент)
- `b` — смещение

**Пример:** Предсказание цены дома на основе площади

```python
# Простой пример
площадь = 100  # м²
вес = 50000    # рублей за м²
смещение = 500000  # базовая цена

цена = вес * площадь + смещение
# цена = 5,500,000 рублей
```

### 2. Логистическая Регрессия

Используется для бинарной классификации (да/нет, 0/1).

**Как работает:**
- Вычисляет вероятность принадлежности к классу
- Использует сигмоидную функцию для ограничения выхода от 0 до 1

**Пример:** Определение спама

```
Вероятность спама = 0.85 → Это спам (порог 0.5)
Вероятность спама = 0.30 → Не спам
```

### 3. Деревья Решений

Строят дерево вопросов для принятия решений.

**Структура:**
```
Доход > 50000?
├── Да → Кредитная история хорошая?
│   ├── Да → ОДОБРИТЬ
│   └── Нет → ОТКАЗАТЬ
└── Нет → ОТКАЗАТЬ
```

**Преимущества:**
- Легко интерпретировать
- Работают с разными типами данных
- Не требуют масштабирования данных

**Недостатки:**
- Склонны к переобучению
- Чувствительны к небольшим изменениям данных

### 4. Метод K-Ближайших Соседей (KNN)

Классифицирует объект по большинству классов среди его соседей.

**Как работает:**
1. Находим K ближайших объектов в обучающей выборке
2. Смотрим, к каким классам они принадлежат
3. Присваиваем наиболее частый класс

**Пример (K=3):**
```
Новый объект окружён: [Красный, Красный, Синий]
→ Классифицируем как Красный
```

### 5. Метод Опорных Векторов (SVM)

Находит оптимальную границу между классами.

**Идея:**
- Найти гиперплоскость, которая максимально разделяет классы
- Максимизировать зазор между ближайшими точками разных классов

### 6. Случайный Лес (Random Forest)

Ансамбль деревьев решений.

**Как работает:**
1. Строим множество деревьев на разных подвыборках
2. Каждое дерево голосует за класс
3. Выбираем класс с большинством голосов

**Преимущества:**
- Высокая точность
- Устойчивость к переобучению
- Работает с большими данными

## 4. Процесс Подготовки Данных

### Этапы работы над проектом МО:

```
1. Сбор данных → 2. Очистка → 3. Исследование → 4. Подготовка → 5. Обучение → 6. Оценка → 7. Внедрение
```

### 1. Сбор данных

**Источники данных:**
- Базы данных компании
- Открытые датасеты (Kaggle, UCI)
- Веб-скрейпинг
- API сервисов
- Сенсоры и устройства IoT

### 2. Очистка данных

**Типичные проблемы:**
- Пропущенные значения
- Дубликаты
- Выбросы (аномальные значения)
- Некорректные форматы
- Ошибки ввода

**Методы очистки:**
- Удаление строк с пропусками
- Заполнение пропусков (среднее, медиана)
- Удаление дубликатов
- Фильтрация выбросов
- Исправление форматов

### 3. Исследовательский анализ данных (EDA)

**Цели:**
- Понять структуру данных
- Выявить закономерности
- Обнаружить проблемы
- Сформулировать гипотезы

**Инструменты:**
- Статистики (среднее, медиана, стандартное отклонение)
- Визуализация (гистограммы, scatter plots, box plots)
- Корреляционный анализ

### 4. Подготовка признаков (Feature Engineering)

**Масштабирование:**
- Приведение признаков к одному диапазону
- Методы: MinMax Scaling, Standardization

**Кодирование категориальных признаков:**
- One-Hot Encoding
- Label Encoding
- Target Encoding

**Создание новых признаков:**
- Комбинация существующих признаков
- Извлечение признаков из даты/времени
- Агрегация данных

### 5. Разделение данных

**Обычное разделение:**
- **Обучающая выборка (Train)**: 60-80% данных
- **Валидационная выборка (Validation)**: 10-20%
- **Тестовая выборка (Test)**: 10-20%

**Важно:** Тестовая выборка должна быть полностью изолирована до финальной оценки!

### 6. Выбор и обучение модели

**Шаги:**
1. Выбрать подходящие алгоритмы
2. Обучить модели на тренировочных данных
3. Оценить на валидационной выборке
4. Настроить гиперпараметры
5. Выбрать лучшую модель

### 7. Оценка модели

**Метрики для классификации:**
- **Accuracy**: доля правильных ответов
- **Precision**: точность (сколько из предсказанных положительных верны)
- **Recall**: полнота (сколько из реальных положительных найдены)
- **F1-Score**: гармоническое среднее Precision и Recall
- **ROC-AUC**: площадь под ROC-кривой

**Метрики для регрессии:**
- **MAE**: средняя абсолютная ошибка
- **MSE**: средняя квадратичная ошибка
- **RMSE**: корень из MSE
- **R²**: коэффициент детерминации

### 8. Внедрение модели

**Этапы:**
1. Сохранение модели
2. Интеграция в продакшен
3. Мониторинг производительности
4. Регулярное переобучение

## 5. Переобучение и Недообучение

### Переобучение (Overfitting)

**Что это:** Модель слишком хорошо запомнила обучающие данные, включая шум.

**Признаки:**
- Высокая точность на обучающих данных
- Низкая точность на тестовых данных

**Как бороться:**
- Упростить модель
- Добавить регуляризацию
- Увеличить количество данных
- Использовать кросс-валидацию
- Применить dropout (для нейронных сетей)

### Недообучение (Underfitting)

**Что это:** Модель слишком проста и не улавливает закономерности.

**Признаки:**
- Низкая точность и на обучающих, и на тестовых данных

**Как бороться:**
- Усложнить модель
- Добавить больше признаков
- Уменьшить регуляризацию
- Обучать дольше

### Идеальный баланс

```
Точность
  ↑
  │     ╱‾‾‾‾‾‾‾‾‾‾ Тестовые данные
  │    ╱
  │   ╱ ‾‾‾‾‾‾‾‾‾‾ Обучающие данные
  │  ╱
  │ ╱
  │╱
  └──────────────────→ Сложность модели
  
       ↓         ↓
  Недообучение  Переобучение
       ↑
  Оптимально
```
